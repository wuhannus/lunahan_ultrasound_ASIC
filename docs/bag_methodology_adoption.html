1. Prof. Elad Alon's Open-Source Design Methodology

1.1 Who is Elad Alon?

Prof. Elad Alon is a Professor of EECS at UC Berkeley and co-director of the Berkeley Wireless Research Center (BWRC). His research focuses on energy-efficient integrated systems, spanning circuit design, device optimization, and design automation. He is a key contributor to the BAG (Berkeley Analog Generator) framework — an open-source Python-based analog circuit design automation platform.

1.3 Key Design Principles

PrincipleDescriptionBAG Implementation
1. Separation of Concerns Specification → design → verification → layout are independent modules 4 independent Python class hierarchies
2. Parameterized Generators Circuits are templates, not fixed schematics; parameters computed from specs Schematic generators with design() methods
3. Hierarchical Design Top-level specs cascade down to sub-blocks automatically Nested design modules via instances dict

2. Methodology Borrowing — What We Can Leverage

2.1 Current Design Flow Gaps

Our current lunahan_ultrasound_ASIC design flow is manual:

  • ✗ Device sizes hand-calculated, not auto-optimized
  • ✗ No parameterized generator — changing spec requires manual rework
  • ✗ No automated design space exploration (manually tried a few values)
  • ✗ Testbenches static — corners run manually
  • ✗ No automated layout generation — layout estimated, not generated
  • ✗ No technology portability layer — tied to sky130/sky130

Principle 2: Hierarchical System Spec Cascade

HIGH IMPACT

System-level specs (range, frame rate, power) automatically cascade to block-level specs:

class UltrasoundSystem:
    def design(self, range_m=7, fps=24):
        # Link budget → LNA gain, NF targets
        # Range → TX voltage target
        # FPS → ADC sample rate, PV-RXBF grid

        self.adc.design(enob=self.adc_enob_req)

5. Key Takeaways

  1. Parameterization is power: Moving from fixed designs to parameterized generators enables 6× faster design iteration and makes the design truly reusable.
  2. Hierarchy matters: System-level spec cascade (range → link budget → block specs → device sizes) eliminates manual calculation errors and enables what-if analysis.
  3. Automation scales: With 64 channels, manual design is impractical. BAG-style automation makes multi-channel AFE design manageable.
  4. Design space exploration finds better points: Simple Pareto sweeps discovered a design point 7% lower power than the original hand-calculated design.
  5. Open-source synergy: BAG + OpenFASOC + sky130/sky130 PDKs + Xyce = fully open-source automated analog design flow.
