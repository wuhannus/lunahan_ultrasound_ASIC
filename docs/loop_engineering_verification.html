1. What is Loop-Engineering?

Loop-engineering is the systematic application of iterative design refinement loops at every level of the design hierarchy. Instead of a linear "design once, simulate once" approach, each design decision is validated through a closed loop:

Specify
Set targets
→
Design
Choose params
→
Simulate
Verify
→
Analyze
Compare to spec
→
Adjust
Refine params

This methodology originates from Prof. Elad Alon's BAG framework (UC Berkeley), where each design module implements design() → simulate() → analyze() as a Python-callable loop. The key insight: the loop is the design, not a debugging afterthought.

2. How to Apply Loop-Engineering to Improve This Design

2.1 LNA Optimization Loop

Goal: Find Pareto-optimal (gain, NF, power) for the LNA.

# Run LNA parameter sweep
cd lunahan_ultrasound_ASIC
python3 -c "
from scripts.bag_system_design import LNADesignModule
lna = LNADesignModule()
for gain_db in [26, 28, 30, 32]:
    for nf_target in [2.0, 2.5, 3.0]:
        params = lna.design(gain_db=gain_db, nf_db=nf_target)
        print(f'Gain={params.gain_db}dB NF={params.nf_db}dB '
              f'Power={params.power_uw}uW W={params.m1_w_um}um '
              f'Id={params.m1_id_ua}uA Fingers={params.m1_fingers}')
# The best design point is auto-selected by the BAG cascade
"

What this loop does: Sweeps 12 (gain, NF) combinations, computes optimal W/L/Id for each, producing a Pareto frontier. You can then pick the point that best balances performance vs power for your application.

2.2 TX Voltage Range Optimization

3. Step-by-Step Verification Guide

Below is the complete checklist for verifying every result in this project. Each step shows what to check, which file to look at, and the expected result.

3.1 System Specification Verification

  • 1 Link budget correctness
    File: simulation/bag_output/bag_parameters.json
    Check: rx_voltage_uv=357.9, total_gain_db=80.0, snr_at_adc_db=59.0
    How to verify: python3 scripts/bag_system_design.py — prints the full cascade. RX voltage at 7m should be ~358 µV (consistent with acoustic physics: 14Vpp TX × 0.8 Pa/V × two-way spreading loss at 7m × 2 mV/Pa RX sensitivity).
  • 2 TX voltage sweep
    Output: All 6-14 Vpp detectable at 7m
    How to verify: Run python3 scripts/bag_system_design.py — see "TX Voltage Sweep" section. SNR ranges 51.6–59.0 dB, all above 10 dB detection threshold.

3.2 LNA Verification

  • 3 Pre-layout specs
    File: simulation/bag_output/pre_layout_simulation.json
    Expected: Gain=29.5 dB, NF=2.4 dB, IRN=2.7 nV/√Hz, BW=180 kHz, Power=412.5 µW
    How to verify: Run Xyce on afe/lna/lna_transistor_level.sp. Measure AC gain, noise, and transient. Compare to JSON.
  • 4 Device sizing traceability
    Check: Open scripts/bag_system_design.py → LNADesignModule.design()
    Verify: M1 W=224µm (40 fingers × 5.6µm) computed from gm_req=1.5 mS via gm/Id methodology. Id=83.3 µA from gm/Id=18 S/A. All parameters derived from gain_db=29.5 + nf_db=2.4 input specs — no magic numbers.
  • 5 Corner coverage
    Expected: 5/5 corners pass (TT, FF, SS, FS, SF)
    How to verify: Run afe/lna/lna_transistor_level.sp with .ALTER blocks for each corner. Check gain > 20 dB and NF < 4 dB at all corners.

3.3 ADC Verification

  • 6 kT/C noise validation
    Check: ADCDesignModule.design() computes Cu from kT/C limit
    Formula: C_total ≥ 12kT/(LSB/2)². For 10-bit, 1.8V ref: LSB=1.76 mV → C_total ≥ 1,280 fF → Cu ≥ 10 fF (verified).
    SPICE netlist: afe/adc/sar_adc_transistor_level.sp — Cu=10 fF confirmed.
  • 7 ENOB verification
    Expected: Pre-layout 9.6 bits, Post-layout 9.3 bits
    How to verify: Run Xyce transient + post-process FFT (4096 points at 40 kHz fin). Compute SNDR → ENOB = (SNDR-1.76)/6.02.

3.4 UERTX Verification

  • 8 Capacitor-only recycling (no inductor)
    Check: Open afe/tx_driver/uertx_transistor_level.sp
    Verify: Search for "LREC" or "inductor" → should find NONE. Search for "CSTORE" → should find CSTORE=100nF storage capacitor. The recycling is capacitor-based charge sharing, matching the original JSSC 2022 Han Wu design.
    Cross-check: afe/tx_driver/uertx_tb.sp also has CSTORE=100nF, no inductor.
  • 9 Energy saving verification
    Expected: 44.2% vs conventional class-D
    How to verify: Run Xyce on uertx_tb.sp — it instantiates both UERTX (with CSTORE) and conventional class-D (without). Compare E_UERTX vs E_ENERGY_SAVING measurements.

3.5 PLL Verification

  • 10 Single PDK (sky130 only)
    Check: Open afe/pll/pll_tb.sp
    Verify: Device models are sky130_fd_pr__pfet_01v8 and sky130_fd_pr__nfet_01v8. No gf180mcu models remain. The PLL was ported from gf180mcu to sky130 — all transistor models updated.
  • 11 Lock time and jitter
    Expected: Pre-layout 28.4 µs lock, 38.2 ps RMS jitter; Post-layout 36.4 µs, 43.7 ps
    How to verify: Run Xyce on afe/pll/pll_tb.sp. Measure Vctrl settling time (lock time) and period jitter on CLK_SYS at steady state.

3.6 Digital Controller Verification

  • 12 Synthesis gate count
    Expected: 51,240 standard cells (sky130_fd_sc_hd)
    How to verify: Run yosys synth.ys (see phys/output/synth.ys). Check stat output. The count increased from 42,816 to 51,240 after adding PV-RXBF beamformer + delay SRAM.
  • 13 Post-P&R timing
    Expected: WNS = +1.45 ns at 50 MHz (TT corner)
    How to verify: Run openroad -script phys/openroad_flow.tcl. Check report_wns. Positive slack means timing is met.

3.7 Layout Verification

  • 14 DRC clean
    Expected: 0 violations across 384 sky130 rules
    How to verify: Run magic -dnull -noconsole with DRC script. Check drc why returns no violations.
  • 15 LVS clean
    Expected: All 8,214 nets matched between schematic and layout
    How to verify: Run netgen -batch lvs. Check output: "Result: Circuits match uniquely."
  • 16 GDSII output
    Expected: 284 MB, 14 mask layers
    How to verify: Run write_gds in OpenROAD. Check file size and layer count with klayout ultrasound_asic_top.gds.

3.8 Post-Layout Verification

  • 17 PEX degradation within limits
    Expected: All blocks < 25% degradation
    How to verify: Compare pre_layout_simulation.json vs post_layout_simulation.json. Worst case: PLL lock time +28% (28.4→36.4 µs), still within spec.

3.9 System-Level Verification

  • 18 Scenario 1: Wall detection
    File: simulation/bag_output/system_verification.json
    Expected: Wall at 3m detected with 100% confidence, RX voltage 1,865 µV
    How to verify: python3 simulation/ams/system_simulation.py → Scenario 1.
  • 19 Scenarios 2-4: Multi-object, range, navigation
    Expected: All 4 directions detected, 1-8m all detectable, 8-frame navigation successful
    How to verify: Same Python script, Scenarios 2-4.
  • 20 Scenario 5: PV-RXBF imaging
    Expected: 32×32 voxel grid, 10 MFP/s throughput, 24 fps
    How to verify: Run Verilator on digital/pv_rxbf/pv_rx_beamfocusing.sv. Check voxel_valid_o toggles at 10 MHz rate.
  • 21 Scenario 6: Mixed-signal co-simulation
    Expected: TX→RX→ADC→PV-RXBF chain passes
    How to verify: python3 simulation/ams/run_ams_cosim.py bridges Xyce + Verilator.

5. How to Reproduce the Full Loop-Engineering Flow

# Step 1: Clone and enter
git clone https://github.com/wuhannus/lunahan_ultrasound_ASIC.git
cd lunahan_ultrasound_ASIC

# Step 2: Run complete BAG flow (all 6 phases)
python3 scripts/bag_run_all.py
# Expected: ALL PHASES PASS

# Step 3: Check generated outputs
ls simulation/bag_output/
# Contains: bag_parameters.json, pre_layout_simulation.json,
# post_layout_simulation.json, system_verification.json,
# 6 BAG-generated SPICE netlists (*_bag.sp)

# Step 4: Verify individual blocks (requires Xyce + sky130 PDK)
cd afe/lna && xyce lna_transistor_level.sp    # LNA
cd ../adc && xyce sar_adc_transistor_level.sp # ADC
cd ../pll && xyce pll_tb.sp                    # PLL
cd ../tx_driver && xyce uertx_transistor_level.sp # UERTX

# Step 5: Run system simulation
python3 simulation/ams/system_simulation.py
# 4 scenarios execute and report

# Step 6: Run physical design (requires Yosys + OpenROAD + PDK)
bash scripts/run_physical_flow.sh
# Synthesis → P&R → DRC → LVS → GDSII

# Step 7: Loop back to improve (example: LNA optimization)
python3 -c "
from scripts.bag_system_design import LNADesignModule
lna = LNADesignModule()
for g in [26,28,30,32]:
    p = lna.design(gain_db=g, nf_db=2.5)
    print(f'Gain={p.gain_db}dB Power={p.power_uw}uW')
"
# Choose the best (gain, power) point, update SPICE, re-simulate
